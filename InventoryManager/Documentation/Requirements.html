Inventory System Requirements

  1. Name: Manager Interface
    Type: Functional
    Summary: User interface accessed by managers. Allows adding/removing/modifying items in the database, create new orders
  2. Name: add_item
    Type: functional
    Summary: Items should be added to data base.
  3. Name: remove_item
    Type: functional
    Summary: Items should be removed to data base.
  4. Name: view_item
    Type: functional
    Summary: Item information should be displayed.
  5. Name: modify_item
    Type: functional
    Summary: Items members should be modifiable.
  6. Name: item_info
    Type: functional
    Summary: Item will contain the following, Name,Price,Product_id,Product_Count,Item_Location
  7. Name: pull_total_items
    Type: functional
    Summary: Must pull total number of individual product
  8. Name: Accept_Order
    Type: Functional
    Summary: Allows manager interface to accept an order from a customer.
    1. Type: Constraint
      Summary: Will ensure sufficient stock on hand to fulfil order
      Note: Need to determine exactly how to handle insufficient stock.
    2. Type: Constraint
      Summary: Convert to a new format to be read by employee interface
    3. Type: Constraint
      Summary: Save converted file in .txt format
  9. Name: Create_Order
    Type: Functional
    Summary: Manager can create an order to restock items
  10. Name: Auto_Order
    Type: Functional
    Summary: System can automatically add items to an order if stock falls below a set threshold
  11. Name: Set_Auto_Order_Threshold
    Type: Functional
    Summary: Set the number that will trigger an auto order when an item stock falls below
  12. Name: Auto_Check_Stock
    Type: Functional
    Summary: Ensures stock does not fall below zero
  13. Name: Stock_Error
    Type: Functional
    Summary: When stock falls below zero an error will be sent to the employee interface
  14. Name: store_data_list
    Type: Risk
    Summary: Must interface with Microsoft Access
  15. Name: Employee Interface
    Type: Functional
    Summary: User interface accessed by employees. Allows viewing items, modifying stock, receiving shipments.
  16. Name: Employee interface
    Type: Functional
    Summary: Employee must have different job screen for each task.
    1. Name: Outgoing Order Processing menu
      Type: Functional
      Summary: All buttons to process orders.
    2. Name: Receving Order Processing menu
      Type: Functional
      Summary: All buttons on receiving orders.
  17. Name: View_order
    Type: Functional
    Summary: In employee interface: Will load a selected order from file
  18. Name: Accept_Order
    Type: Functional
    Summary: Load order to Employee interface.
  19. Name: Complete_Order
    Type: Functional
    Summary: Removes order from orders to be completed, and updates stock
