Inventory System Requirements

  1. Name: Manager Interface
    Type: Functional
    Summary: User interface accessed by managers. Allows adding/removing/modifying items in the database, create new orders
  2. Name: Employee Interface
    Type: Functional
    Summary: User interface accessed by employees. Allows viewing items, modifying stock, receiving shipments.
  3. Name: add_item
    Type: functional
    Summary: Items should be added to data base.
  4. Name: remove_item
    Type: functional
    Summary: Items should be removed to data base.
  5. Name: view_item
    Type: functional
    Summary: Item information should be displayed.
  6. Name: modify_item
    Type: functional
    Summary: Items members should be modifiable.
  7. Name: item_info
    Type: functional
    Summary: Item will contain the follow, Name,Price,Product_id,Product_Count,Item_Location
  8. Name: pull_total_items
    Type: functional
    Summary: Must pull total number of individual product
  9. Name: Accept_Order
    Type: Functional
    Summary: Allows manager interface to accept an order from a customer.
    • Type: Constraint
      Summary: Will only accept .txt files
    • Type: Constraint
      Summary: Will ensure sufficient stock on hand to fulfil order
      Note: Need to determine exactly how to handle insufficient stock.
    • Type: Constraint
      Summary: Convert to a new format to be read by employee interface
    • Type: Constraint
      Summary: Save converted file in .txt format
  10. Name: store_data_list
    Type: risk
    Summary: Must interface with Microsoft Access
